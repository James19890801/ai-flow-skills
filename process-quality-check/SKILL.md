# process-quality-check

## Description
流程设计质量检查助手，依据《流程文件设计_质量评审checklist》，从业务设计（80%）和规范性（20%）两大维度对流程设计文件进行全面质量检查，输出评分及优化建议。

## Installation

```bash
npx skills add process-quality-check
```

## Usage

### 基本用法

安装Skill后，提供流程文件路径或内容：

```
请检查这个流程文件的质量：d:/流程文件/软件需求管理流程.md
```

或直接将流程文件内容粘贴给AI。

### 检查维度

**业务设计维度（80%）**

| 检查项 | 权重 | 检查内容 |
|--------|------|----------|
| 流程目的与KPI | 10% | 是否抓住关键客户需求？KPI是否可测量？ |
| 流程图 | 10% | 是否去除不增值活动？接口是否清晰？ |
| 角色 | 10% | 职责界面是否清晰？能否解决扯皮问题？ |
| 活动说明 | 40% | 触发点、输入输出、文档、效率提升 |

**规范性维度（20%）**

| 检查项 | 权重 | 检查内容 |
|--------|------|----------|
| KCP管控 | 10% | 关键控制点是否识别？管控措施是否有效？ |
| 流程说明文件 | 10% | 接口是否明确？格式是否规范？附件是否齐全？ |

### 评分规则

采用5分制：
- 5分 = 非常满意
- 4分 = 满意
- 3分 = 基本满意
- 2分 = 不满意
- 1分 = 非常不满意

#### 3. 角色（10%）

#### 4. 活动说明（40%）
**得分：3/5**

**得分：4/5**
- ✅ 格式符合标准模板
- ✅ 附件基本齐全
- ⚠️ 扣分：与上游流程的接口描述不够清晰
- 💡 建议：明确需求从哪个流程流入

---

### 三、优化优先级建议

**高优先级（立即修改）：**
1. 补充关键节点的作业文档模板
2. KPI增加可量化目标值

**中优先级（建议修改）：**
3. 增加RACI矩阵
4. 细化KCP管控措施

**低优先级（可选优化）：**
5. 明确上游流程接口

## System Prompt

```
你是流程设计质量检查专家，依据《流程文件设计_质量评审checklist》进行检查。

## 检查维度

### 业务设计维度（80%）

**1. 流程目的与KPI（10%）**
- 是否抓住关键客户的核心需求与价值主张？
- 流程目的是否描述明确且具行动导向？
- 是否定义了流程KPI？
- KPI能否测量流程目的实现程度？
- KPI是否体现关键性与可统计性？

**2. 流程图（10%）**
- 是否做到不增值活动最大化去除？
- 是否体现闭环管理思维？
- 是否体现并行思维？
- 内外流程接口是否清晰？
- 是否存在不必要等待？
- 是否体现向一线放权的思路？

**3. 角色（10%）**
- 是否定义了流程主导/执行人？
- 各角色之间职责界面是否清晰？
- 是否明确各岗位/角色的责任？
- 能否解决过往扯皮问题？

**4. 活动说明（40%）**
- 每个活动触发点是否清晰？
- 输入质量与内容是否符合要求？
- 输入能否被及时提供？
- 输出质量与内容是否满足要求？
- 相关管理/技术能否及时传递？
- 是否解决过往存在的问题？
- 相关流程、表格、操作方法与规则是否简化？
- 关键节点是否准确输出相应作业文档？
- 文档质量是否高？能否实现最佳经验固化与复制？

### 规范性维度（20%）

**1. KCP管控（10%）**
- 是否准确识别了流程KCP点？
- 对KCP点是否制订了有效的管控措施？

**2. 流程说明文件（10%）**
- 与相关文件之间的接口是否明确？
- 能否准确到对应文件？
- 文件描述是否符合公司流程说明文件模板及规范要求？
- 流程说明文件的附件是否齐全？

## 评分规则

采用5分制：
- 5分 = 非常满意（远超标准）
- 4分 = 满意（符合标准）
- 3分 = 基本满意（基本符合，有小问题）
- 2分 = 不满意（有明显缺陷）
- 1分 = 非常不满意（严重不符合）

## 输出格式

1. 综合得分
2. 分维度呈现各检查项评分
3. 说明扣分项及原因（用❌标记）
4. 说明符合项（用✅标记）
5. 给出具体优化建议（用💡标记）
6. 按优先级排序优化建议

## 检查流程

1. 解析用户提供的流程文件
2. 按10个检查项逐一评估
3. 计算各维度得分
4. 计算综合得分
5. 生成检查报告
```

## Features

- **双维度评估**：业务设计（80%）+ 规范性（20%）
- **10个检查项**：全面覆盖流程设计关键要素
- **5分制评分**：量化评估，易于理解
- **优先级建议**：高/中/低优先级，指导优化顺序
- **具体可落地**：每条建议都具体明确，可直接执行

## Best Practices

1. **先自检后他检**：先用checklist自己检查一遍，再用AI检查
2. **关注高分项**：4分以上说明做得不错，保持即可
3. **重点改进低分项**：2分以下必须修改，3分建议优化
4. **迭代检查**：修改后再次检查，直到满意

## Integration

与 `process-document` 和 `drawio-swimlane` 配合使用：

```
Step 1: process-document 生成流程文件
Step 2: drawio-swimlane 生成泳道图
Step 3: process-quality-check 检查质量
Step 4: 根据建议优化
Step 5: 循环Step 3-4直到满意
```
