# 流程挖掘与诊断 Skill

## 核心交付物

1. **动态流程轨迹可视化**：青色流动粒子沿审批路径实时流动，节点大小=平均耗时，红色脉冲标识卡点
2. **深度诊断报告（HTML）**：包含语义分析、AI 辅助优化方案、行业对标、改进建议的完整交互式报告

## 输入数据要求

用户可提供以下格式的流程数据：

- **JSON**：结构化流程实例数据
- **Excel/CSV**：表格形式的流程轨迹数据

### 关键数据维度

| 维度 | 字段示例 | 用途 |
|------|----------|------|
| **流程标识** | 实例 ID、流程类型 | 区分不同流程实例 |
| **节点信息** | 节点名称、节点类型、处理人/角色 | 构建流程拓扑 |
| **时间数据** | 节点开始时间、结束时间、整体耗时 | 识别耗时异常节点 |
| **审批结果** | 通过、驳回、转办、挂起 | 分析流程健康度 |
| **驳回信息** | 驳回节点、驳回原因、重审路径 | 定位卡点 |
| **审核意见** | 备注、意见文本 | 语义分析，发现共性问题 |

## 鲁棒性保障原则（最高优先级）

> 任何步骤失败都不能导致整体中断，必须降级处理后继续输出。

### 数据容错规则

| 异常情况 | 处理策略 |
|----------|----------|
| 数据格式非JSON/Excel | 尝试按纯文本/CSV解析；仍失败则请求用户重新提供并说明支持格式 |
| 关键字段缺失（如无时间字段）| 跳过依赖该字段的分析步骤，在报告中注明"因缺少时间数据，耗时分析不可用" |
| 数据量不足（<10条）| 发出警告但继续分析，报告中标注"样本量较小，结论仅供参考" |
| 字段名与预期不符 | 执行模糊匹配（如 "time"/"duration"/"耗时" 均映射到时间字段），匹配结果告知用户确认 |
| 审批意见字段为空 | 跳过语义分析模块，不影响其他模块输出 |
| 数值字段含非数值 | 自动过滤/转换，记录清洗行数，在报告中说明 |
| 编码问题（乱码）| 尝试 UTF-8、GBK、GB2312 依次解码；告知用户检测到的编码 |

### 输出降级策略

当部分数据/分析缺失时，报告**必须仍然输出**，缺失模块用灰色占位卡片替代，标注"数据不足，跳过此分析"，不留空白。

## 执行流程

```
用户上传数据 (JSON/Excel/CSV/文本) 
    ↓
【步骤 1】数据解析与结构识别
    - 自动识别数据格式（JSON/Excel/CSV/纯文本）
    - 格式识别失败 → 尝试多种解析器；均失败则明确告知支持格式并中止
    - 提取关键字段映射（支持模糊匹配字段名）
    - 验证数据完整性，记录缺失字段
    ↓
【步骤 2】数据完整性检查
    - 检查关键字段是否存在（时间/节点/结果）
    - 部分字段缺失 → 记录缺失清单，继续执行（降级模式）
    - 数据量不足（<10条）→ 发出警告但不中断
    - 字段存在但数据质量差 → 清洗后继续，报告清洗情况
    - 仅当无法提取任何有效数据时 → 反问用户补充
    ↓
【步骤 3】流程轨迹构建
    - 解析每个实例的完整轨迹
    - 计算节点停留时长
    - 统计流转频次
    - 识别循环（驳回重审）路径
    ↓
【步骤 4】卡点/堵点/断点识别
    - 耗时异常节点（超过 P90 阈值；数据不足时降级为均值+标准差方法）
    - 高驳回率节点
    - 断点（流程中断/挂起）
    - 热点路径（高频流转）
    - 无法计算时：在报告该区域标注"数据不足以识别卡点"
    ↓
【步骤 5】审批意见语义分析
    - 关键词提取与分类（薪资/职级/编制/经验等）
    - 高频问题词云
    - 典型意见样本展示
    - 意见字段为空 → 跳过此步骤，报告对应区域显示占位提示
    ↓
【步骤 6】AI 辅助优化方案生成
    - 规则引擎自动化机会识别
    - AI 预审能力矩阵
    - 智能决策辅助建议
    - 数据不足时仍基于流程结构给出通用建议，标注"基于流程结构推断"
    ↓
【步骤 7】深度联网搜索
    - 搜索类似场景的行业最佳实践
    - 学术论文中的流程优化方法
    - 咨询公司报告（麦肯锡、德勤等）
    - 专家公众号文章
    - 对标分析
    - 联网失败 → 跳过此步骤，依靠内置知识库给出建议，报告中注明"未启用联网搜索"
    ↓
【步骤 8】构建数据结构（关键步骤）
    - 构建 nodeData 数据结构：
      ```javascript
      const nodeData = {
        '节点名称': { avg:平均耗时, max:最大耗时, count:经过案例数, type:'节点类型', layer:0, col:列位置 },
      };
      ```
    - 构建 edgeData 数据结构：
      ```javascript
      const edgeData = [
        { from:'起始节点', to:'目标节点', flow:流转频次, isMain:true/isRework:true/isStop:true },
      ];
      ```
    - 节点类型：start(起始)/task(任务)/approval(审批)/bottleneck(卡点)/end(结束)/stop(终止)/rework(重审)
    - 路径类型：isMain(主路径)/isRework(重审路径)/isStop(终止路径)/isAbnormal(异常跳转)
    ↓
【步骤 9】生成可视化轨迹图
    - 分层布局（从左到右：发起→审批→结束）
    - **节点大小动态计算**：
      ```javascript
      const r = minR + (avg / maxAvg) * (maxR - minR);
      // 节点半径 = 最小半径 + (平均耗时/最大平均耗时) * (最大半径-最小半径)
      ```
    - **边线宽=流转频次**：
      ```javascript
      const thick = 1 + (flow / maxFlow) * 4;
      // 连线粗细根据流转频次动态计算
      ```
    - 青色流动粒子动画（可播放/暂停/调速）
    - 节点颜色 = 类型（蓝=起止/绿=任务/橙=审批/红=卡点）
    - 卡点红色脉冲动画
    - 悬停显示详细指标（平均耗时、最大耗时、经过案例数、节点类型）
    - 可视化生成异常 → 降级输出静态流程摘要表格，不中断报告
    ↓
【步骤 10】生成诊断报告
    - 执行摘要（核心指标卡片）
    - 动态流程可视化（或降级静态表）
    - 流程路径发现（主路径+变体路径+驳回终止路径）
    - 审批意见语义分析（有数据则展示，无则显示占位）
    - AI 辅助优化方案
    - 核心洞察与改进建议
    - 数据质量说明（清洗了多少条、跳过了哪些模块）
    ↓
【步骤 11】浏览器预览（最后一步）
    - 启动本地HTTP服务器：python -m http.server 8080
    - 调用 run_preview 工具打开浏览器预览
    - 或使用 run_in_terminal 打开系统默认浏览器
    ↓
交付：单一 HTML 页面（完整交互式报告）+ 浏览器预览
    - HTML 生成异常 → 降级输出结构化 Markdown 报告
```

## 数据结构规范（核心要点）

### 节点数据结构 (nodeData)

```javascript
const nodeData = {
  '合同起草':     { avg:112, max:174, count:72, type:'start',      layer:0, col:0 },
  '总经理审批':   { avg:1523, max:2865, count:62, type:'bottleneck', layer:0, col:6 },
  '合同归档':     { avg:76,  max:120, count:63, type:'end',        layer:0, col:8 },
  '终止':         { avg:0,   max:0,   count:9,  type:'stop',       layer:-1, col:4.5 },
};
```

| 字段 | 说明 | 示例 |
|------|------|------|
| avg | 平均停留耗时（分钟） | 1523 |
| max | 最大停留耗时（分钟） | 2865 |
| count | 经过该节点的案例数 | 62 |
| type | 节点类型 | bottleneck |
| layer | 层级（0=主路径，-1=异常路径） | 0 |
| col | 列位置（从左到右） | 6 |

### 边数据结构 (edgeData)

```javascript
const edgeData = [
  // 主路径
  { from:'合同起草', to:'法务初审', flow:72, isMain:true },
  // 驳回终止路径
  { from:'总经理审批', to:'终止', flow:2, isStop:true },
  // 重审路径
  { from:'HR预审', to:'HR预审（重审）', flow:1, isRework:true },
];
```

| 字段 | 说明 | 示例 |
|------|------|------|
| from | 起始节点名称 | '合同起草' |
| to | 目标节点名称 | '法务初审' |
| flow | 流转频次 | 72 |
| isMain | 是否主路径 | true |
| isRework | 是否重审路径 | true |
| isStop | 是否终止路径 | true |
| isAbnormal | 是否异常跳转 | true |

### 节点大小计算公式

```javascript
// 节点半径根据平均耗时动态计算
const maxAvg = 1523; // 所有节点中的最大平均耗时
const minR = 22, maxR = 55; // 最小/最大半径（像素）
const r = minR + (avg / maxAvg) * (maxR - minR);

// 示例：总经理审批 avg=1523, 则 r = 22 + (1523/1523)*(55-22) = 55 (最大)
// 示例：合同归档 avg=76, 则 r = 22 + (76/1523)*(55-22) ≈ 23.6 (较小)
```

### 边线宽计算公式

```javascript
// 连线粗细根据流转频次动态计算
const maxFlow = 72; // 最大流转频次
const thick = 1 + (flow / maxFlow) * 4;

// 示例：主路径 flow=72, 则 thick = 1 + 4 = 5 (最粗)
// 示例：终止路径 flow=2, 则 thick = 1 + (2/72)*4 ≈ 1.1 (较细)
```

## 可视化设计规范

### 配色方案

- **主色调**：青蓝紫渐变 (#0ea5e9 → #6366f1 → #8b5cf6)
- **流动粒子**：青色 (#22d3ee) + 三层柔和光晕
- **背景**：深蓝渐变 (#0f172a → #1e293b)
- **节点类型**：
  - 开始/结束：蓝色 (#3b82f6)
  - 任务节点：绿色 (#10b981)
  - 审批节点：橙色 (#f59e0b)
  - 卡点/堵点：红色 (#ef4444) + 脉冲动画

### 布局规范

- **分层布局**：按流程阶段从左到右排列
  - 第 1 层：流程发起
  - 第 2 层：HR 提交申请
  - 第 3 层：部门负责人审批 / HR 经理审批（上下分叉）
  - 第 4 层：薪酬确认
  - 第 5 层：VP 审批 / CEO 审批（上下分叉，高 P 级）
  - 第 6 层：发送 Offer
  - 第 7 层：候选人确认
  - 第 8 层：流程结束

### 动态效果

- **流动粒子**：
  - 大小：6px
  - 颜色：青色 + 柔和光晕
  - 速度：可调节（0.3x - 5x）
  - 密度：根据实际流转频次调整透明度

- **卡点脉冲**：红色呼吸灯效果，2 秒周期

- **悬停交互**：节点放大 1.1 倍，显示详细 Tooltip

### 控制面板

- 开始/暂停按钮（带状态指示器）
- 加速/减速按钮
- 重置按钮
- 实时状态显示（运行中/已停止）

## 诊断报告结构

### 1. 执行摘要

- 流程实例数、轨迹总数
- 平均耗时、审批意见数
- AI 优化项数量
- 核心指标卡片（悬停动效）

### 2. 动态流程轨迹可视化

- 分层布局流程图
- 青色流动粒子动画
- 播放/暂停/调速控制
- 节点悬停详情
- 图例说明

### 3. 审批意见语义分析

- **问题分类统计**：薪资/职级/编制/经验/学历/稳定性/流程
- **高频关键词云**：带频次标签
- **典型审批意见**：原文展示 + 自动分类标签

### 4. AI 辅助优化方案

- **规则引擎自动化**：高置信度场景（通过率>90%）
  - 场景描述
  - 置信度
  - 平均耗时
  - 年节省工时估算

- **AI 预审能力矩阵**：
  - 问题类型
  - AI 解决方案
  - 实施难度（低/中/高）

### 5. 核心洞察与改进建议

- 卡点定位（如：CEO 审批是最大堵点）
- 驳回率分析（如：候选人确认环节流失率高）
- 规则化潜力评估
- 机审结合点建议

## 反问用户的问题模板

当数据不足以支撑结论时，按以下优先级反问：

1. **数据完整性**
   - "当前数据缺少【节点耗时】字段，能否补充？"
   - "审核意见字段为空，是否有其他渠道获取？"

2. **业务背景**
   - "这个流程的业务目标是什么？"
   - "当前流程 SLA 要求是多少？"
   - "哪些节点是系统审批，哪些是人工审批？"

3. **问题聚焦**
   - "您最关注哪个环节的优化？"
   - "历史是否做过流程优化？效果如何？"

## 输出格式

单一 HTML 文件，包含：

- 嵌入式 CSS（青蓝紫渐变主题、响应式布局）
- 原生 JavaScript 可视化（流动粒子动画、交互控制）
- 结构化诊断内容（语义分析、AI 方案、核心洞察）
- 专业深色主题设计

## 技术实现要点

1. **可视化引擎**：原生 JavaScript + CSS，不依赖外部库（D3.js 可选）
2. **动画性能**：使用 requestAnimationFrame 确保流畅
3. **响应式设计**：适配不同屏幕尺寸
4. **交互细节**：
   - 按钮真实可用（非占位）
   - 状态实时反馈
   - Tooltip 精确定位
   - 悬停动效流畅
5. **HTML 容错机制**：
   - 所有 JS 代码用 try/catch 包裹关键计算逻辑
   - Canvas/SVG 初始化失败时降级为纯 DOM 表格展示
   - 避免使用 `document.write()`，避免同步阻塞渲染
   - 动画循环用 `cancelAnimationFrame` 保证清理，防内存泄漏

## 关键技术要点

1. **节点大小=平均耗时**：耗时越长，节点越大，一眼识别卡点
2. **边线宽=流转频次**：流转越多，连线越粗，识别热点路径
3. **真实流转路径**：从数据中提取，不假设线性流程，含驳回终止、重审循环、异常跳转
4. **hover显示详情**：悬停节点显示平均耗时、最大耗时、经过案例数、节点类型
5. **卡点红色脉冲**：bottleneck类型节点显示红色脉冲动画
6. **浏览器预览**：生成HTML后必须启动服务器并打开浏览器预览

## 注意事项

1. **客观性**：所有结论必须有数据支撑，避免主观臆断
2. **可操作性**：建议必须具体、可落地，避免空泛
3. **优先级**：按影响面和实施难度排序建议
4. **可视化优先**：复杂数据用图表表达，降低阅读成本
5. **行业对标**：必须联网搜索验证，确保建议的先进性
6. **交互真实**：所有按钮必须真实可用，状态反馈及时
7. **最后一步**：必须通过终端启动HTTP服务器并打开浏览器预览
