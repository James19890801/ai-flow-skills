# 流程挖掘与诊断 Skill

## 核心交付物

1. **动态流程轨迹可视化**：青色流动粒子沿审批路径实时流动，节点大小=平均耗时，红色脉冲标识卡点
2. **深度诊断报告（HTML）**：包含语义分析、AI 辅助优化方案、行业对标、改进建议的完整交互式报告

## 输入数据要求

- **JSON**：结构化流程实例数据
- **Excel/CSV**：表格形式的流程轨迹数据

### 关键数据维度

| 维度 | 字段示例 | 用途 |
|------|----------|------|
| 流程标识 | 实例ID、流程类型 | 区分不同流程实例 |
| 节点信息 | 节点名称、类型、处理人 | 构建流程拓扑 |
| 时间数据 | 开始时间、结束时间、耗时 | 识别耗时异常节点 |
| 审批结果 | 通过、驳回、转办、挂起 | 分析流程健康度 |
| 审核意见 | 备注、意见文本 | 语义分析发现共性问题 |

## 执行流程

步骤1 数据解析 → 步骤2 完整性检查 → 步骤3 轨迹构建 → 步骤4 卡点识别 → 步骤5 语义分析 → 步骤6 AI优化方案 → 步骤7 联网搜索 → 步骤8 可视化生成 → 步骤9 输出诊断报告

## 可视化设计规范

- **主色调**：青蓝紫渐变 (#0ea5e9 → #6366f1 → #8b5cf6)
- **背景**：深蓝渐变 (#0f172a → #1e293b)
- **卡点**：红色脉冲动画
