# 流程智能监控设计 Skill

## 核心交付物

1. **《流程智能监控方案》**：含指标定义、阈值设置、异常分类、根因分析框架、优化决策树、Owner报告模板的完整结构化文档
2. **可视化监控看板（HTML文件）**：基于分析数据生成交互式HTML监控看板，包含实时指标展示、趋势图表、异常预警面板
3. **预警报告示例**：模拟一份Owner会收到的分析报告样本，帮助理解方案落地效果

## 鲁棒性保障规则（最高优先级）

> 用户提供信息不完整时，不能中断或报错，必须基于已有信息智能补全，并标注"AI推断"。

| 异常情况 | 处理策略 |
|----------|----------|
| 用户只说流程名，没有其他信息 | 基于流程名推断常见痛点和指标，生成草案，告知用户可以修正 |
| 关键节点未提供 | 基于流程类型（审批/运营/服务）推断3-5个典型节点，标注"AI推断，请确认" |
| 无 SLA 目标 | 使用行业基准值并标注来源，如"合同审批行业基准：3-5个工作日" |
| 用户描述模糊 | 不要求澄清所有信息，先出完整方案，在报告末尾列出"建议确认的问题" |
| 步骤1信息采集无响应 | 超过1轮无有效输入后，直接进入方案生成，用最保守的通用模板 |

## 执行流程

```
用户描述流程名称与业务目标
    ↓
【步骤 1】信息采集
    - 流程名称与所属业务域
    - 流程核心目标（如：合同审批 → 目标：3天内完成审批）
    - 关键节点（2-5个主要审批/处理节点）
    - 当前已知的痛点（可选）
    ↓
【步骤 2】指标设计
    - 推荐2-4个核心监控KPI
    - 每个指标含：名称 / 定义 / 计算方式 / 正常范围 / 异常阈值
    - 优先推荐：时效类指标 + 质量类指标 + 异常类指标
    ↓
【步骤 3】异常规则设置
    - 超时类异常：超过时限X天/小时触发预警
    - 高驳回异常：驳回率超过阈值触发分析
    - 重复异常：同一实例被驳回≥N次触发深度分析
    - 断点异常：流程中断/挂起超过X小时触发报警
    - 用户可自定义补充规则
    ↓
【步骤 4】根因分析框架
    - 针对每类异常，给出标准排查思路
    - 例：高超时率 → 排查：节点处理人负荷 / 信息不完整 / 流程规则问题
    ↓
【步骤 5】优化决策树
    根据异常类型，给出决策路径：
    - 改规则（调整审批条件/阈值）
    - 改流程（调整节点顺序/增减节点）
    - 授权（增加审批人/设置代理规则）
    - 培训（执行者能力问题 → 培训）
    - AI赋能（识别AI可介入的环节）
    ↓
【步骤 6】AI赋能机会识别（核心增值环节）
    基于异常分析结果，识别AI可介入的高价值场景：
    
    【识别逻辑】
    - 驳回率高 → 驳回前置：提交前AI校验，把问题拦截在发起环节
    - 审批滞留长 → AI辅助审批：帮助审批人快速审核，减少停留时长
    - 规则明确且风险低 → AI自动审批：省掉审批节点，直接通过
    - 重复驳回多 → 检查是否有标准化模板缺失
    
    【AI赋能三维度】
    ┌─────────────────────────────────────────────────────────────┐
    │ 维度一：驳回前置 - 把驳回拦在提交环节                        │
    │ 适用场景：驳回率高、驳回原因集中在规则性问题                   │
    │ AI动作：提交前校验 → 发现问题 → 提示用户修正                  │
    ├─────────────────────────────────────────────────────────────┤
    │ 维度二：AI辅助审批 - 帮审批人快速做决策                       │
    │ 适用场景：审批人滞留时间长、信息复杂需要汇总                   │
    │ AI动作：汇总关键信息 → 给出审批建议 → 审批人一键确认           │
    ├─────────────────────────────────────────────────────────────┤
    │ 维度三：AI自动审批 - 直接省掉审批节点                         │
    │ 适用场景：规则明确、风险低、历史通过率高                       │
    │ AI动作：自动判断 → 符合条件直接通过 → 异常才转人工            │
    └─────────────────────────────────────────────────────────────┘
    
    【场景匹配】根据异常类型匹配AI赋能维度（详见后文「AI赋能场景库」）
    - 合同审批类 → 驳回前置(条款预审) + AI辅助审批(法务审核)
    - 费用报销类 → 驳回前置(标准校验) + AI自动审批(低金额)
    - 采购申请类 → AI辅助审批(需求汇总) + AI自动审批(常规采购)
    - 招聘审批类 → 驳回前置(编制校验)
    - 付款审批类 → 驳回前置(发票校验) + AI自动审批(合同已归档)
    
    【输出要求】
    - 必须明确AI赋能维度（驳回前置/辅助审批/自动审批）
    - 场景名称 + 痛点关联 + AI能力说明 + 预期效果
    - 必须给出具体的落地步骤和数据准备清单
    - 按优先级排序（见效快、难度低的优先推荐）
    ↓
【步骤 7】输出监控方案
    生成完整《流程智能监控方案》文档
    ↓
【步骤 8】生成可视化监控看板（HTML）
    基于分析数据生成交互式HTML监控看板，包含：
    - 核心指标卡片（平均时长、超时率、异常率、通过率）
    - 流程类型分布图表
    - 节点瓶颈分析可视化
    - 异常预警列表面板
    - 趋势分析图表
    ↓
【步骤 9】生成Owner预警报告模板
    基于方案，生成一份模拟的Owner预警通知示例
    ↓
【步骤 10】自动打开看板预览
    使用PowerShell命令自动打开HTML看板：
    - 命令：Start-Process "看板文件完整路径"
    - 无需用户确认或额外指令，确保交付即可见
```

## 标准监控指标体系

### 时效类指标
| 指标 | 定义 | 建议阈值参考 |
|------|------|------------|
| 平均审批时长 | 从提交到最终完成的平均天数 | 根据SLA定义 |
| 超时率 | 超过约定时限的实例占比 | >20%触发预警 |
| 节点滞留时长 | 单个节点平均等待时间 | >SLA的50%触发预警 |

### 质量类指标
| 指标 | 定义 | 建议阈值参考 |
|------|------|------------|
| 驳回率 | 被驳回实例占总数比例 | >30%触发异常 |
| 重复驳回率 | 同一实例被驳回≥2次的比例 | >10%触发深度分析 |
| 一次通过率 | 首次提交即通过的比例 | <60%需关注 |

### 异常类指标
| 指标 | 定义 | 建议阈值参考 |
|------|------|------------|
| 周驳回频次 | 同一流程一周内被驳回次数 | ≥3次触发异常标记 |
| 断点率 | 流程中断/挂起实例占比 | >5%触发排查 |
| 异常集中度 | 异常是否集中在特定节点/人员/时段 | 集中度>60%需定向分析 |

## 可视化监控看板（HTML）输出规范

当用户提供流程轨迹数据时，**必须**生成交互式HTML监控看板文件。

### HTML看板必备组件

| 组件 | 内容要求 | 图表类型 |
|------|----------|----------|
| **核心指标卡片区** | 平均处理时长、超时率、异常率、一次通过率 | 数字卡片+趋势箭头 |
| **流程类型分布** | 各流程类型占比、数量 | 饼图或柱状图 |
| **业务域分布** | 各业务域流程数量 | 横向条形图 |

| **异常预警列表** | 高风险流程、异常实例明细 | 表格+状态标签 |
| **SLA达成情况** | 各SLA等级的达成率 | 仪表盘或环形图 |

### HTML技术规范

- **框架**：使用原生HTML+CSS+JavaScript，不依赖外部框架

- **样式**：响应式布局，适配不同屏幕尺寸
- **交互**：支持图表hover提示、数据筛选、时间范围切换
- **文件命名**：`[流程名称]_智能监控看板.html`

## 流程治理最佳实践（内置知识）

### 异常定义原则
1. **驳回异常**：同一流程实例一周内被驳回≥3次，定义为异常流程，需人工介入
2. **超时分级**：超时≤50%为黄色预警（催办），超时>100%为红色报警（升级处理）
3. **节点集中预警**：同一节点在同一时段异常数量>整体的60%，触发节点专项分析

### Owner收到的应该是报告，不是数字
- 错误方式：推送"超时率：38%"
- 正确方式：推送"合同审批超时率本周达38%，法务节点为主要卡点，建议授权二级法务处理标准合同"

### 优化决策优先级
1. 先排查规则问题（最容易改，成本最低）
2. 再看流程设计问题（需要重新建模）
3. 最后考虑执行者问题（需要培训）
4. 有条件时考虑自动化（高置信度场景）

## 反问用户的问题模板

当信息不足时，优先级顺序：

1. **流程基本信息**
   - "这个流程的核心业务目标是什么？（比如：3天内完成审批）"
   - "流程有哪几个主要节点？"

2. **当前痛点**
   - "目前这个流程最让人头疼的问题是什么？"
   - "有没有已知的卡点或高频问题？"

3. **Owner情况**
   - "谁是这个流程的负责人？他/她希望收到什么格式的报告？"

## AI赋能场景库（核心知识）

> 💡 **AI赋能三维度**：不是只有「驳回前置」一种方式，而是三个方向：
> - **维度一：驳回前置** - 把驳回拦在提交环节
> - **维度二：AI辅助审批** - 帮审批人快速做决策
> - **维度三：AI自动审批** - 直接省掉审批节点

---

### 【维度一】驳回前置场景

> 适用：驳回率高、驳回原因集中在规则性问题
> 效果：驳回率大幅降低

#### 场景1：费用报销智能校验
**痛点**：财务驳回率>25%，超预算、超标报销、票据不合规
**AI赋能类型**：🔴 驳回前置

| 检查项 | AI能力 | 触发时机 |
|-------|--------|--------|
| 费用标准校验 | 对照费用标准表，识别超标项 | 提交时实时反馈 |
| 票据OCR识别 | 自动提取发票信息，校验抬头、金额、日期 | 上传附件时 |
| 预算余额校验 | 实时查询部门预算余额 | 提交时 |

**落地步骤**：数字化费用标准 → 对接OCR → 配置校验规则
**预期效果**：财务驳回率降低50%

---

#### 场景2：招聘编制前置校验
**痛点**：驳回率>40%，编制未开放、薪资超预算
**AI赋能类型**：🔴 驳回前置

| 检查项 | AI能力 | 数据来源 |
|-------|--------|--------|
| 编制状态校验 | 实时查询岗位编制状态 | HR系统 |
| 薪资预算校验 | 对比岗位薪酬band + 部门预算余额 | 薪酬系统 |

**落地步骤**：打通编制/薪酬系统 → 配置校验规则 → 提交环节前置校验
**预期效果**：编制/预算问题驳回减少80%

---

#### 场景3：合同智能预审
**痛点**：驳回率>40%，条款问题、资质问题
**AI赋能类型**：🔴 驳回前置

| 检查项 | AI能力 | 输出示例 |
|-------|--------|--------|
| 价格条款核验 | 对比历史合同价格、市场价格区间 | ⚠️ 价格偏离度+35% |
| 风险条款识别 | 匹配风险条款库，标记异常条款 | 🔴 发现排他性条款 |
| 对方资质校验 | 对接企查查/天眼查API | ⚠️ 对方有经营异常 |

**落地步骤**：建立条款库 → 对接外部数据 → 配置预审规则
**预期效果**：驳回率降低50%

---

### 【维度二】AI辅助审批场景

> 适用：审批人滞留时间长、信息复杂需要汇总
> 效果：审批时长大幅缩短

#### 场景4：合同审批AI辅助
**痛点**：法务审核节点平均停留6-8小时，需要逐条核对条款
**AI赋能类型**：🟡 AI辅助审批

| AI能力 | 具体动作 | 效果 |
|-------|---------|------|
| 条款汇总 | 自动提取合同关键条款，生成摘要表 | 节省80%阅读时间 |
| 风险标注 | 高亮风险条款，标注风险等级 | 一眼定位问题 |
| 审批建议 | 基于历史审批，给出「建议通过/建议修改」 | 辅助决策 |
| 对比分析 | 与标准合同对比，标注差异点 | 快速识别偏差 |

**落地步骤**：
1. 建立合同审批决策辅助界面
2. 集成条款分析和风险标注功能
3. 训练审批建议模型（基于历史数据）

**预期效果**：法务审核时长从6-8小时降至**1-2小时**

---

#### 场景5：采购审批AI辅助
**痛点**：部门经理/副总审批节点平均停留40+小时，需要了解需求背景
**AI赋能类型**：🟡 AI辅助审批

| AI能力 | 具体动作 | 效果 |
|-------|---------|------|
| 需求汇总 | 自动汇总采购需求、规格、预算、供应商信息 | 一页看清 |
| 预算对比 | 对比历史采购价格、市场价格区间 | 判断合理性 |
| 库存提示 | 显示相关库存情况，提示是否有替代品 | 避免重复 |
| 审批建议 | 基于规则给出「建议批准/需关注」 | 辅助决策 |

**落地步骤**：
1. 开发审批辅助界面
2. 集成预算、库存、历史价格数据
3. 配置审批建议规则

**预期效果**：审批时长从40+小时降至**8小时以内**

---

#### 场景6：招聘审批AI辅助
**痛点**：VP审批节点平均停留24+小时，需要了解岗位背景
**AI赋能类型**：🟡 AI辅助审批

| AI能力 | 具体动作 |
|-------|---------|
| 岗位画像 | 汇总岗位需求、薪资预算、编制状态 |
| 市场对比 | 对比市场薪资水平，标注是否合理 |

## AI赋能落地方法论

### 三维度优先级判断

| AI维度 | 适用痛点 | 落地难度 | 见效速度 | 推荐优先级 |
|-------|---------|---------|---------|----------|
| 🔴驳回前置 | 驳回率高(>30%) | 低-中 | 快(1-2周) | ⭐⭐⭐⭐⭐ 优先做 |
| 🟡辅助审批 | 审批滞留长(>24h) | 中 | 中(1个月) | ⭐⭐⭐⭐ 其次做 |
| 🟢自动审批 | 规则明确、风险低 | 中-高 | 慢(2-3月) | ⭐⭐⭐ 最后做 |

### 优先级判断矩阵
| 维度 | 评分标准 |
|-----|---------|
| 痛点严重程度 | 驳回率>40% = 3分，20-40% = 2分，<20% = 1分 |
| 落地难度 | 规则类（低）= 3分，需对接系统（中）= 2分，需组织调整（高）= 1分 |
| 见效速度 | 1周内 = 3分，1个月内 = 2分，季度内 = 1分 |

**优先级 = 痛点严重程度 × 落地难度 × 见效速度**

### 落地三阶段
```
Phase 1：驳回前置（1-2周）| 优先级最高
├─ 费用报销智能校验 → 驳回前置
├─ 招聘编制前置校验 → 驳回前置
└─ 特点：规则清晰、数据已有、见效快

Phase 2：AI辅助审批（1个月）
├─ 合同审批AI辅助 → 辅助审批
├─ 采购审批AI辅助 → 辅助审批
└─ 特点：需开发审批辅助界面、集成多源数据

Phase 3：AI自动审批（2-3个月）
├─ 低金额费用报销自动审批 → 自动审批
├─ 常规采购自动审批 → 自动审批
└─ 特点：需灰度验证、风险控制、组织授权调整
```

### 关键成功要素
1. **数据准备**：规则数字化、知识库建设、系统对接
2. **规则先行**：先明确业务规则，再让AI执行
3. **灰度发布**：AI结果先「建议」模式，验证准确率后再「拦截」或「自动」
4. **反馈闭环**：人工审批结果回流训练AI模型
5. **风险控制**：自动审批必须有兜底机制（异常转人工）

## 注意事项

1. **方案要可落地**：所有指标和阈值必须结合用户的实际业务场景，不能照搬模板数字
2. **Owner视角**：始终从流程Owner的角度设计报告，他/她收到的应该是"建议"而不是"数据"
3. **循序渐进**：建议先监控2-3个最关键指标，跑通之后再扩展，不要一次设计10个指标
4. **AI赋能是核心增值**：监控不是目的，改善才是目的。每个监控报告必须包含可落地的AI赋能建议
5. **不混淆流程挖掘**：本Skill聚焦于设计监控规则和AI赋能建议，不做历史数据的回溯分析（那是process-mining的工作）
6. **必须有HTML输出**：当用户提供数据时，必须生成可视化HTML看板，不能只输出文字报告
