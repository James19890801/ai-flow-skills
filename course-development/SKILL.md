# course-development

## Description
AI赋能流程管理课程开发技能，用于为零技术背景的业务/流程管理人员设计并交付约1小时的实战课程。课程以学员动手实操为核心，配套现代化可交互PPT、技能封装与双端同步。

## 鲁棒性保障规则（最高优先级）

| 异常情况 | 处理策略 |
|----------|----------|
| 课程主题描述模糊 | 基于最近似的流程管理场景自动推断，生成草案，末尾列出"建议确认的问题"，不要求先澄清 |
| pptxgenjs 不可用/未安装 | 降级为生成详细的 Markdown 内容稿，告知用户可手动导入到 PowerPoint |
| PPT 生成脚本报错 | 输出完整 Markdown 内容稿作为替代交付物，不中断整体流程 |
| 双端同步一端失败 | 继续同步到另一端，报告哪一端失败和原因，提示手动复制的路径 |
| 文件写入失败 | 在对话窗口输出完整内容，提示用户手动保存 |

## Installation

```bash
npx skills add course-development
```

## Usage

### 基本用法

告诉AI你要开发的课程主题：

```
帮我开发一节关于【AI赋能流程挖掘诊断】的课，面向流程管理人员。
```

AI将按以下流程自动完成：
1. 澄清课程受众与目标
2. 拆解核心交付物（学员动手做什么）
3. 规划内容结构（极简理论 + 大量实操）
4. 生成PPT内容稿（Markdown格式）
5. 生成可交互PPT（pptxgenjs生成.pptx）
6. 封装配套Skills（双端同步）
7. 自动打开预览

## 课程设计四大原则

### 原则一：面向零技术背景学员（强制）
- ❌ 禁讲：参数配置、代码逻辑、技术实现细节
- ✅ 只讲：概念是什么、原理是什么、怎么用
- ✅ 类比优先：用日常生活类比解释技术概念
  - 如：Skill = 手机APP，装上就能用
  - 如：Qoder = AI的工作台，就像专业工具箱

### 原则二：实操为主，理论极简（强制）
- 总时长：约60分钟
- 理论讲解：≤15分钟（概念+演示）
- 学员动手：≥45分钟
- 每节课学员必须带走具体交付物

### 原则三：课件面向学员直出（强制）
- ❌ 禁止在PPT中出现：时间分配说明、教学设计意图、内部思考过程
- ✅ 只呈现：学员需要看到的知识内容
- 课件风格：现代化AI科技感，可交互、可点击
- 使用pptxgenjs生成.pptx文件

### 原则四：技能双端同步（强制）
- 本地路径：D:/Skills/
- 云端路径：D:/AI赋能流程管理课程/02-课程Skills/
- 每次交付后必须同步到两个位置

## 标准课程结构（12页模板）

| 页码 | 内容 | 要点 |
|------|------|------|
| 第1页 | 封面 | 标题、副标题、讲师信息 |

## 理论框架页设计规范（强制）

### 为什么需要理论框架页？
- 建立课程的专业高度和认知基础
- 用图形化方式呈现抽象概念
- 让学员一开始就理解"这是什么"和"为什么"

### 设计原则
- **必须图形化**：禁止纯文字，必须用模型/框架/流程图
- **高度抽象**：提炼课程核心理论，不是细节操作步骤
- **视觉冲击力**：占据整页核心区域，配色突出

### 常见理论框架类型

| 课程类型 | 推荐框架 | 示例 |
|---------|---------|------|
| 流程洞察类 | 金字塔模型 | 数据层→洞察层→行动层 |
| 流程运营类 | 飞轮模型 | 定义指标→设置规则→预警推送→优化迭代 |
| 流程建模类 | 分层架构 | 业务层→逻辑层→执行层 |
| AI通识类 | 认知罗盘 | 四象限展示能力边界 |

### 实现方式
```javascript
// 金字塔模型示例
s.addShape(pptx.ShapeType.roundRect, { x: 3.5, y: 1.2, w: 6.0, h: 1.4, fill: { color: C.accent3 }, rectRadius: 0.15 });
s.addText('行动层', { ... }); // 顶层

s.addShape(pptx.ShapeType.roundRect, { x: 2.5, y: 2.8, w: 8.0, h: 1.4, fill: { color: C.accent }, rectRadius: 0.15 });
s.addText('洞察层', { ... }); // 中层

s.addShape(pptx.ShapeType.roundRect, { x: 1.5, y: 4.4, w: 10.0, h: 1.4, fill: { color: C.bg2 }, rectRadius: 0.15 });
s.addText('数据层', { ... }); // 底层
```

## PPT设计规范

### 配色方案（AI科技感）
```
primary:   #1a1a2e  // 深蓝黑（背景主色）
secondary: #16213e  // 深蓝（背景次色）
accent:    #667eea  // 亮蓝紫（强调色）
accent2:   #764ba2  // 紫色（辅助强调）
text:      #ffffff  // 白色（主文字）
textMuted: #a0a0a0  // 灰色（次要文字）
highlight: #f39c12  // 橙色（特别强调）
```

### 版式规范
- 标题：36-44pt，Microsoft YaHei Bold
- 正文：14-16pt，Microsoft YaHei
- 卡片式布局：圆角矩形，带色彩边框
- 交替深蓝/深蓝黑背景，形成节奏感
- 每页必须有视觉元素（卡片/图标/流程图）

## 课程内容设计规范

### 概念解释三步法
1. **一句话定义**：用最简单的语言说清楚是什么
2. **生活类比**：找一个学员熟悉的场景做类比
3. **价值说明**：说清楚对学员有什么好处

**示例：**
```
Skill是什么？
→ 一句话：AI的专业技能包，装上就会做专业的事
→ 类比：就像手机APP，装了微信才能聊天
→ 价值：装上drawio-swimlane，AI立刻会画专业流程图
```

### 实操任务设计规范
- 任务必须与学员日常工作直接相关
- 提供2-4个可选题目（不强制指定）
- 输出物必须是学员能带走使用的文件
- 讲师走动辅导，而非集体讲授

### Q&A预设规范
每节课至少预设4个常见问题：
- 1个关于AI输出质量
- 1个关于工具使用
- 1个关于需求描述
- 1个关于文件查找/管理
